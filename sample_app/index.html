What is this?

This is a demonstration of how FullStory's session replay URLs can be fused with sentiment anaylsis on customer feedback to automatically generate issues in GitHub.

How can I test it out?

  1. Click the Feedback tab parked over there on the right-hand side of this screen ⇒
  2. Type something unfriendly (feedback with negative sentiment will create a GitHub issue)
  3. Send your feedback
  4. Checkout the GitHub issues list for this project: https://github.com/patrickbrandt/fullstory-demo/issues
  5. You'll see a new open issue with a "negative-feedback" label that contains your feedback and a FullStory session replay URL

What about feedback that isn't negative?

I've got you covered. You can find all feedback and FullStory session replay URLs here: https://wpb.is/FeedbackPortal

What's the tech stack?

  • Polymer 3.0 for the feedback widget
  • React for the feedback portal
  • The Serverless Framework with AWS API Gateway, AWS Lambda, and DynamoDB for the integration API
  • AWS Comprehend for sentiment analysis
  • GitHub Apps and the GitHub REST API for creating issues on GitHub
  • The FullStory JavaScript API for Session Replay URLs

How can I watch the replays?

Right now all session replay URLs are tied to my FullStory account. However, code for everything outlined on this page is on GitHub at https://github.com/patrickbrandt/fullstory-demo. Feel free to fork it and update all account ids (for FullStory and GitHub) to make it your own.
